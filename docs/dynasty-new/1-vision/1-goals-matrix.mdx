This page is the **north star** for all design decisions in Hoop Dynasty.

Every mechanic, system, and trade-off should ultimately serve one or more of the five pillars below.
When a decision is made, it is tagged against these pillars in `6-meta/goals.yaml` with an impact score.
The chart visualises where weight has been intentionally placed (**intent**) and where it has actually landed based on the decisions made so far (**actual**).

## Priority matrix

<GoalsMatrix pillars={pillars} colors={colors} />

The **arc width** of each inner segment reflects the pillar's *intent weight* — how much design importance it was consciously assigned.
The **coloured tick mark** on the inner ring shows where the *actual weight* falls based on scored design decisions.

| Colour | Meaning |
|---|---|
| Green tick | Actual ≈ intent (within 1.5 pp) |
| Orange tick | Actual > intent (over-represented in decisions) |
| Blue tick | Actual < intent (under-represented in decisions) |

## Alignment summary

export const pillarTotals = pillars.map(p => {
  const score = decisions
    .filter(d => d.pillars.includes(p.id))
    .reduce((s, d) => s + d.impact, 0);
  return { ...p, score };
});

export const grandTotal = pillarTotals.reduce((s, p) => s + p.score, 0);

| Pillar | Decision score | Actual % | Intent % | Gap |
|---|---:|---:|---:|---:|
{pillarTotals.map(p => {
  const actualPct = ((p.score / grandTotal) * 100).toFixed(1);
  const gap = (parseFloat(actualPct) - p.intent_weight).toFixed(1);
  const sign = parseFloat(gap) > 0 ? '+' : '';
  return `| ${p.name} | ${p.score} | ${actualPct}% | ${p.intent_weight.toFixed(1)}% | ${sign}${gap}pp |`;
}).join('\n')}
